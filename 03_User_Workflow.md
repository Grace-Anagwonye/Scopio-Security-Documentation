Scopio Security – Detailed User Workflow (Including Live Lectures)

This document outlines the complete user journey on the Scopio Security platform, covering self-paced learning, hands-on labs, live teaching sessions, assessments, and career support throughout the 6-month training program.


 
1. User Accesses the Platform:

New User Actions:

- Opens the Scopio Security website or app.
- Navigates to “Register” or “Sign up".

System Actions:

- Loads authentication page.
- Displays available programs and learning tracks.


2. Registration and Account Creation

User Actions:

- Enters full name, email, phone number, and password.
- Chooses account type (Individual / Corporate).
- Selects preferred training track.
- Accepts terms and conditions.

System Actions:

- Validates details.
- Creates user account.
- Sends verification email or SMS.
- Redirects user to verification page.



3. Email/Phone number Verification:

User Actions:

- Enters verification code.
- Confirms account.

System Actions:

- Matches code.
- Activates account.
- Redirects user to login page.



4. First Login + Onboarding:

User Actions:
- Logs in with email and password
- Views onboarding walkthrough explaining:
  • Lab environment
  • Live classes
  • Program calendar
  • Dashboard layout
- Completes profile setup.

System Actions:

- Loads personalized dashboard.
- Displays upcoming live session schedule.
- Recommends learning modules.



5. Program Structure Overview Display:

When the user enters their dashboard, they see a **6-month blended training structure**.

It includes:

- Weekly live classes with instructors (3 sessions per week).
- Self-paced video modules to study with outside live sessions.
- Hands-on cyber labs.
- Real-world attack simulations.
- Module assessments per week unlocked after completing training for each week.
- Capstone SOC project at the end.

System Actions:

- Loads program calendar.
- loads already recorded modules.
- Shows next live session countdown.
- Displays module progression status.



6. Weekly Live Class Participation

User Actions:

- Views schedule of live classes for the week.
- Joins live sessions through video conferencing (Zoom, Google Meet, or embedded video tool depending on the preference of student and instructor).
- Interacts with instructor (questions, screen sharing, chat).
- Completes in-class tasks or demonstrations.


System Actions:

- Sends reminders before class.
- Opens live class room link.
- Tracks attendance.
- Saves lecture recordings for replay.


What live lecturers do:

- Teach cybersecurity concepts for the week.
- Demonstrate tools used in real SOC environments.
- Explain lab tasks before students attempt them.
- Goes through assessment done by the student per week for scoring and corrections, if any.
- Hold revision sessions and Q&A.
  


7. Self-Paced Video Learning:

 User Actions:

- Opens weekly video modules.
- Watches lectures, tutorials, and tool demonstrations.
- Marks videos as completed.

 System Actions:
 
- Tracks viewing progress.
- Unlocks next module once conditions are met.
- Syncs progress with overall program timeline.



8. Hands-On Lab Practice:

  User Actions:
  
- Launches lab environments from dashboard.
- Reads detailed instructions for each exercise.
- Performs tasks such as:
  • Reconnaissance  
  • Exploitation  
  • Threat identification  
  • Log analysis  
  • Malware detection  
  • Defensive configurations  

System Actions:

- Starts sandboxed virtual environment.
- Tracks live user activity.
- Measures accuracy, time taken, commands entered.
- Generates lab score at the end of each lab practice.




9. Weekly Assignments or Mini-Projects:

User Actions:

- Downloads or opens weekly assignments.
- Completes tasks involving research, screenshots, or lab actions.
- Uploads submission.


System Actions:

- Records submission.
- Sends for grading or automated scoring.




10. Real-World Attack/Defense Simulations:

User Actions:

- Navigates to the simulation dashboard.
- Starts a scenario such as:
  • Ransomware attack simulation  
  • Network intrusion investigation  
  • Phishing attack analysis  
  • SOC alert handling  

System Actions:

- Generates alerts, logs, warnings.
- Tracks user investigation path.
- Scores performance.
- Unlocks next difficulty level.



11. Assessments & Knowledge Checks:

User Actions:

- Attempts quizzes, timed tests, or practical assessments.
- Reviews feedback and weak areas.

System Actions:

- Grades test automatically.
- Unlocks next modules.
- Updates performance analytics.



12. Capstone Project (Mini SOC Setup):


User Actions:

- Enters the 6th-month capstone phase.
- Sets up and operates a mini Security Operations Center (SOC).
- Monitors real-time logs and alerts.
- Investigates attack incidents.
- Writes full incident report.
- Submits project to instructors.

System Actions:

- Generates SOC environment.
- Tracks all activities.
- Stores project submission.
- Sends project to instructor for manual review.



13. Career Support Stage:

User Actions:

- Uploads resume.
- Books mock interview sessions.
- Receives improvement feedback.
- Joins career Q&A sessions with mentors.


System Actions:

- Stores updated resume versions.
- Schedules and logs mock interviews.
- Sends notifications and reminders to students.


14. Completion & Certification Guidance:

User Actions:

- Successfully completes all modules, labs, and capstone.
- Downloads completion certificate.
- Log request for assistance for CEH, CISSP, or Security+ exam registrations.

System Actions:

- Marks program as completed.
- Generates completion badge/certificate.
- Updates user profile.



15. Logout and Re-Entry

User Actions:

- Logs out.
- Returns at any time to:
  - Rewatch videos  
  - Review lecture recordings  
  - Access labs  
  - Download certificates  


System Actions:

- Saves last activity.
- Reloads personalized dashboard on next login.
